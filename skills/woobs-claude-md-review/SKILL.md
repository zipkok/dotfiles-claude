# Woobs Claude MD Review

현재 프로젝트의 `CLAUDE.md`와 `.claude/skills/*/SKILL.md` 파일이 공식 규칙에 따라 잘 작성되었는지 검토합니다.

## 사용법

- `/woobs-claude-md-review` — 현재 디렉터리 기준으로 전체 검사 실행

## 동작 방식

다음 순서로 실행합니다:

1. **파일 탐색** — 현재 작업 디렉터리에서 아래 파일을 찾습니다
   - `CLAUDE.md`
   - `.claude/skills/*/SKILL.md` (하위 디렉터리 포함)
2. **존재 여부 확인** — 각 파일이 없으면 ❌ 로 표시하고 다음으로 넘어갑니다
3. **파일 읽기** — 존재하는 파일을 읽습니다
4. **체크리스트 검토** — 아래 체크리스트 기준으로 각 항목을 검토합니다
5. **결과 출력** — 통과(✅) / 미흡(⚠️) / 누락(❌) 으로 출력하고 개선 제안을 제공합니다

---

## CLAUDE.md 체크리스트

**필수**
- [ ] 파일이 존재하는가
- [ ] 프로젝트 목적/설명이 명확하게 작성되어 있는가
- [ ] Claude가 코드를 읽어도 알 수 없는 정보(빌드 커맨드, 환경 변수, 아키텍처 결정 등)가 포함되어 있는가

**권고**
- [ ] 200줄 이하인가 (초과 시 Claude adherence 저하)
- [ ] Claude가 이미 알고 있는 표준 관행, 파생 가능한 정보(파일 경로, git 히스토리 등)를 중복 기록하고 있지 않은가
- [ ] 자주 변경되는 정보나 긴 튜토리얼이 포함되어 있지 않은가
- [ ] 200줄 초과 시 `@path` 임포트 또는 `.claude/rules/` 로 분리했는가
- [ ] 마크다운 헤더와 불릿 포인트로 구조화되어 있는가

---

## SKILL.md 체크리스트

**필수**
- [ ] 파일이 존재하는가
- [ ] H1 제목(스킬 이름)이 있는가

**권고 (frontmatter)**
- [ ] YAML frontmatter(`---`)가 있는가
- [ ] `description` 필드가 있는가 (Claude 자동 로드 여부 결정에 사용)
- [ ] 인자를 받는 경우 `argument-hint` 필드가 명시되어 있는가
- [ ] 필요한 경우 `allowed-tools`로 사용 도구가 제한되어 있는가

**권고 (내용)**
- [ ] 스킬의 목적이 명확하게 설명되어 있는가
- [ ] 수행할 동작이 구체적으로 기술되어 있는가
- [ ] `$ARGUMENTS` 등 변수 활용이 적절한가
- [ ] 500줄 이하인가

## 출력 형식

```
## 검사 결과

### CLAUDE.md
| 항목 | 결과 |
|------|------|
| 파일 존재 | ✅ |
| 목적/설명 | ✅ |
| Claude 지시사항 | ⚠️ |
| 200줄 이하 | ✅ |
| 파생 정보 중복 없음 | ✅ |

### .claude/skills/idea/SKILL.md
| 항목 | 결과 |
|------|------|
| 파일 존재 | ✅ |
| H1 제목 | ✅ |
| frontmatter | ❌ |
| 동작 방식 | ✅ |
| 500줄 이하 | ✅ |

### 개선 필요 항목
| 구분 | 파일 | 코드라인 | 항목 | 문제 | 제안 |
|------|------|----------|------|------|------|
| ❌ 필수 | idea/SKILL.md | `L1` | frontmatter | 없음 | 파일 최상단에 --- frontmatter 추가 |
| ⚠️ 주의 | CLAUDE.md | `-` | Claude 지시사항 | 없음 | 지시사항 또는 컨텍스트를 추가하세요 |

## 이후 작업 제안
N개 중 N개 통과. ❌ 필수 항목을 먼저 수정하고, ⚠️ 주의 항목은 선택적으로 개선하세요.
```
