# AI 사용 코칭

사용자가 Claude를 더 잘 쓰도록 돕는 메타 피드백. 모든 세션에서 자동 로드된다.

## 출력 조건 (게이트)

**현재 비활성화됨 (2026-05-26 사용자 요청)** — 어떤 프롬프트에도 코칭 블록을 출력하지 않는다. 아래 평가 프레임워크·출력 형식·테이블 규칙은 참고용으로 보존되며, 다시 활성화하려면 이 게이트 섹션을 원래 내용으로 되돌린다.

## 평가 프레임워크 — 4원칙

[forrestchang/andrej-karpathy-skills CLAUDE.md](https://github.com/forrestchang/andrej-karpathy-skills/blob/main/CLAUDE.md)의 4원칙을 "사용자 프롬프트가 이 원칙을 잘 활성화했나"로 뒤집어 평가한다.

| 원칙 | 코칭 질문 |
|------|----------|
| **Think Before Coding** | 가정·모호함이 잘 드러났나? 다중 해석을 미리 차단/허용했나? |
| **Simplicity First** | 요구 범위가 적절했나? 추측성·과잉 요청은 없었나? |
| **Surgical Changes** | 변경 스코프가 명확했나? 인접 영역까지 "정리"를 유도하진 않았나? |
| **Goal-Driven Execution** | 검증 가능한 성공 기준이 있었나? "잘 해줘"가 아니라 "X면 통과"였나? |

## 출력 형식

평가가 어렵거나 대상이 없는 항목은 생략하지 말고 `–` 또는 `해당없음`으로 표기해 일관성을 유지한다.

```
---
### 💡 AI 사용 코칭

- **이번 프롬프트**
  - [어떻게 요청했는지 1줄, 없으면 "해당없음"]

- **원칙 진단**

  ```
  +-----------------------+------+
  | Principle             | Eval |  Comment
  +-----------------------+------+
  | Think Before Coding   |  ✅  |  [한 줄, 해당없으면 "–"]
  | Simplicity First      |  ✅  |  [한 줄]
  | Surgical Changes      |  ✅  |  [한 줄]
  | Goal-Driven Execution |  ✅  |  [한 줄]
  +-----------------------+------+
  ```

- **기타 관찰**
  - [4원칙 외 패턴 — 컨텍스트 양·예시 제공·스코프 명시·도구 선택·반복 가능성 등. 없으면 "해당없음"]

- **개선 제안**
  - [가장 약한 원칙 또는 관찰을 토대로 다음에 시도할 1줄. 없으면 "해당없음"]
```

기호 가이드: ✅ 잘 활성화 / ⚠️ 부분적·모호 / ❌ 누락·반대 방향 / `–` 해당없음(평가 대상 아님).

## 테이블 작성 규칙

- 코드 펜스(```) 안에 ASCII 테이블로 출력 → 마크다운 렌더링과 무관하게 하단 경계까지 항상 보임
- **고정폭 컬럼**(Principle, Eval)은 ASCII 영문만 사용해 정렬을 보장한다. 한글·이모지(double-width)는 정렬을 깨므로 좌측 컬럼에 넣지 말 것
- **Comment는 우측 경계 밖**에 배치 → 길이가 자유로워 한글/이모지 사용 가능, 정렬 부담 없음
- 상단·하단 `+---+` 경계선을 반드시 닫아 테이블이 완결되었음을 표시
- 코멘트가 길면 줄바꿈 대신 요약 (한 줄 원칙 유지)

같은 조언 반복 금지 — 새로운 패턴이 보일 때만 가치 있다. 별도 로그 파일은 만들지 않는다(대화 안에서만 표시).
